# IV-трейсер (ВАХ) и библиотека компонентов

Страница **«ВАХ»** снимает вольт-амперную характеристику двухполюсника — строит ток
против напряжения, кривую I(V) — затем анализирует её, позволяет накладывать
несколько кривых для сравнения и ведёт библиотеку охарактеризованных деталей.
Вкладки: **Свип** (старт / живой), **Профили**, **История**, **Библиотека**,
**Сравнение**.

![IV-трейсер](../screenshots/iv-tracer.png)

## Снятие кривой (F-024)

Свип — это telemetry-driven step-loop: на каждом шаге трейсер пишет уставку, ждёт
осевшего чтения и записывает **измеренную** точку `(V, I)`. DPS-150 — источник с
лимитом **compliance**: токовым (для свипа напряжения) или вольтовым (для свипа
тока), который защищает исследуемый компонент (DUT). Каждая записанная точка — это
пересечение кривой DUT с лимитом, то есть настоящая точка ВАХ компонента.

### Пресеты компонентов

| Компонент | Свип | Compliance | Анализ |
|---|---|---|---|
| **LED** | 0 → 6 В | 20 мА | Vf @ 20 мА, идеальность *n*, apparent Rs, rd |
| **Диод** (1N400x) | 0 → 1 В | 100 мА | Vf @ 100 мА (колено), *n*, apparent Rs, rd |
| **Zener** | 0 → Vz + ~20 % | из Pmax/Vz | Vz @ Izt, Zzt (подключать реверсом) |
| **Резистор** | 0 → √(P·R) | Vmax/R | R, R², макс. отклонение |
| **Лампа** | 0 → номинал | ~1.5× номин. ток | R холодная, R горячая, отношение |

### Шаги

1. **Выберите пресет** (или сохранённый профиль) на вкладке «Свип» — он заполнит
   диапазон свипа, compliance и число шагов для этого типа компонента. Подключите
   DUT к клеммам (Zener — **реверсом**, катод к +).
2. **Старт** — включение выхода подтверждается. Выход энергизируется с уже
   выставленным compliance, и живая кривая I(V) строится точка за точкой.
3. По завершении откройте свип (История, Библиотека или живой вью), чтобы увидеть
   кривую, затенённую полосу compliance и панель **«Анализ»**.

### Честный анализ — без выдуманных чисел

Реальные свипы ломают наивный фиттинг, поэтому **каждая метрика nullable**. Когда
значение нельзя посчитать надёжно, приложение показывает **«—»** с причиной в
*Notes*, а не уверенно-неверное число:

- **Непроводящий** DUT (обрыв, реверс, `vStop` ниже Vf) → `did-not-conduct`, без
  метрик.
- Идеальность **n** фитится только по экспоненциальной области (отбирается по
  измеренному току, исключая шумовой пол и clamp'нутые compliance точки); при
  нехватке точек остаётся «—», а даже посчитанная помечается **приближённой**
  (линейный свип напряжения недосэмплирует декаду).
- **Zener**, не достигший пробоя → `breakdown-not-reached`, без Vz.
- Последовательное сопротивление помечено **apparent** (завышает истинный Rs).

На скриншоте выше идеальность честно равна «—» («too few in-region points»), тогда
как Vf и apparent Rs показаны.

## Сравнение свипов (F-025)

Вкладка **«Сравнение»** накладывает несколько кривых I(V) на один график. Попасть
туда можно тремя путями: мульти-выбор свипов в **Истории**, выбор нескольких
компонентов в **Библиотеке** или «Сравнить все свипы» у компонента. Выбор живёт в
URL (`?tab=compare&ids=…`), поэтому сравнение **bookmarkable и шарится**.

![Сравнение кривых I(V)](../screenshots/iv-compare.png)

- Кривые рисуются сырыми, авто-фит на объединение, с тумблером **линейный / лог Y**
  (лог — правильная линза для диодов/LED), легендой с показ/скрыть по кривой и до
  ~8 кривых.
- Когда все выбранные свипы **одного типа компонента**, появляется **таблица
  сравнения метрик** со столбцом min / max / **разброс** — идеально для матчинга
  или биннинга партии LED или диодов. Null'ы игнорируются (никогда не становятся
  NaN).
- **Экспорт CSV** скачивает все выбранные кривые в одном длинном файле
  (`sweepId, label, index, voltage, current, power`).

## Библиотека компонентов (F-025)

Вкладка **«Библиотека»** — каталог охарактеризованных физических деталей (имя, тип,
партномер, заметки). Привяжите один или несколько свипов к компоненту и закрепите
один как **эталонный** — его сохранённые метрики и кривая становятся канонической
характеристикой компонента.

![Библиотека компонентов](../screenshots/iv-library.png)

Используйте её, чтобы собрать эталон детали для стенда, отследить компонент по
температурам или «Сравнить все свипы», чтобы увидеть его разброс. Удаление
компонента никогда не удаляет его свипы; эталонный пин самозалечивается, если
закреплённый свип отвязать.

> **Никаких солнечных / PV-кривых.** ВАХ фотоэлемента (Pmax, Vmp, Imp, fill-factor)
> снимают, нагружая *освещённую* ячейку, пока она отдаёт мощность — DPS-150 —
> однoквадрантный источник и не может потреблять фототок ячейки, поэтому этот
> квадрант физически недостижим. *Тёмный* прямой свип ячейки — это просто диодный
> анализ выше.
