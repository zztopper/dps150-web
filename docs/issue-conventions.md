# Issue Conventions

Issues в GitLab/GitHub — **единственный источник истины** по бэклогу,
техдолгу и багам. `.claude/plans/` — только оперативное планирование внутри сессии.

## Нумерация

Формат заголовка: `PREFIX-NNN: Название`.

| Префикс | Label              | Описание                  |
|---------|--------------------|---------------------------|
| `F-`    | `feature`          | Новая функциональность    |
| `B-`    | `bug`              | Баг / дефект              |
| `TD-`   | `tech-debt`        | Технический долг          |
| `S-`    | `security`         | Безопасность              |
| `D-`    | `documentation`    | Документация              |
| `I-`    | `infra`            | Инфраструктура / DevOps   |

Нумерация **сквозная внутри класса**, без пропусков. Первый — `F-001`.

Получить следующий номер:

```bash
./scripts/issue-counter.sh F   # -> F-042
```

## Жизненный цикл

1. **Новый план** → создаём Issue, привязываем к Milestone, ставим labels.
   - Обязательная RICE-оценка тремя агентами (`docs/rice-scoring.md`).
2. **Старт работы** → исполнитель назначен, статус `In Progress`.
3. **Завершение** → закрываем Issue с комментарием, который содержит:
   - ссылку на MR/PR;
   - краткий список ключевых коммитов;
   - заметки по техдолгу, если что-то отложено.
4. **Изменение скоупа** → актуализируем Issue, при необходимости создаём
   связанный Issue (label `tech-debt` или новый `F-`).
5. **Найден баг или техдолг по ходу** → отдельный Issue (`B-` или `TD-`),
   в комментарии оригинального Issue ссылка.

## Labels

Минимальный набор:
- по типу: `feature`, `bug`, `tech-debt`, `security`, `documentation`, `infra`;
- по подсистеме: `backend`, `frontend`, `bot`, `db`, `ci`, `docs`;
- статусы (опционально, GitLab board): `status::backlog`, `status::in-progress`, `status::review`, `status::blocked`.

## Milestones

Группировка по итерациям. Активные открыты, завершённые закрыты.
Шаблон: `Этап N: <Название>` (`Этап 1 PoC`, `Этап 2 MVP`, `Этап 3 v1.0`).

## Ссылки между Issues

- В описании MR: `Closes #NNN` (GitHub) или `Closes !NNN` / `Related to #NNN` (GitLab).
- В комментарии Issue: `Связано с F-XXX, блокируется TD-YYY`.

## Helper-скрипты

| Скрипт                            | Что делает                                            |
|-----------------------------------|-------------------------------------------------------|
| `scripts/issue-counter.sh PREFIX` | Возвращает следующий номер для префикса.              |
| `scripts/issue-create.sh ...`     | Создаёт Issue с правильным заголовком и labels.       |
| `scripts/issue-list.sh ...`       | Список Issues по статусу/labels.                      |
| `scripts/rice-template.sh ...`    | Печатает шаблон описания с RICE-таблицей.             |

Все скрипты работают и с GitLab (через `glab`), и с GitHub (через `gh`) —
выбирается через `ISSUE_PROVIDER` в `.env`.

## Template описания Issue

См. вывод `./scripts/rice-template.sh "Название"`. Минимальные блоки:
1. Бизнес-контекст.
2. Acceptance Criteria.
3. Out of scope.
4. Технические детали.
5. RICE-таблица (три голоса + среднее).
